## 1. 호이스팅(Hoisting)이란?

코드 실행 전, 자바스크립트 엔진이 변수와 함수 선언문을 스크립트의 **최상단으로 끌어올리는 것처럼** 동작하는 현상을 말한다. 실제 코드가 이동하는 것이 아니라, 엔진이 실행 준비 단계에서 선언문을 먼저 메모리에 등록하기 때문에 발생한다.

---

## 2. 호이스팅이 가능한 이유: 2단계 실행 과정

자바스크립트 엔진은 코드를 실행하기 전 반드시 두 가지 단계를 거친다.

### ① 컴파일 단계 (Creation Phase)

코드를 한 줄씩 실행하기 전, 전체 코드를 훑으며 **선언문**을 찾아낸다.

* **메모리 할당**: 함수 선언문과 변수 선언문을 발견하면 이를 **실행 컨텍스트** 내의 메모리 공간에 미리 등록한다.
* **초기화**: `var`로 선언된 변수는 이 단계에서 `undefined`로 초기화됩니다. (함수 선언문은 함수 객체 전체가 메모리에 올라간다.)

### ② 실행 단계 (Execution Phase)

실제 코드를 위에서부터 한 줄씩 실행하는 단계이다.

* **값 할당**: 컴파일 단계에서 확보된 메모리 공간에 실제 값을 대입하거나 함수를 호출한다.
* **참조**: 선언문보다 앞에서 변수를 호출하더라도, 이미 컴파일 단계에서 메모리에 등록되어 있기 때문에 오류 없이 접근이 가능하다.

---

## 3. 선언 방식에 따른 차이

| 구분 | 호이스팅 발생 여부 | 특징 |
| --- | --- | --- |
| **var** | O | 선언과 동시에 `undefined`로 초기화되어 접근 가능 |
| **function** | O | 함수 전체가 호이스팅되어 선언 전 호출 가능 |
| **let / const** | O (실질적 제한) | 호이스팅은 되지만 초기화 전까지 **TDZ(Temporal Dead Zone)**에 머물러 접근 시 에러 발생 |

---

## 4. 요약

* 자바스크립트 엔진은 **선언(컴파일)**과 **할당(실행)**을 분리해서 처리한다.
* 컴파일 단계에서 선언문을 메모리에 미리 올려두기 때문에 호이스팅이 가능해진다.
* 이는 코드의 유연성을 제공하지만, `var`의 경우 예상치 못한 `undefined` 참조를 유발할 수 있으므로 주의가 필요하다.
