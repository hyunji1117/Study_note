## 1. 개념 정의

### **px (Pixel)**
화면의 물리적인 **최소 단위(픽셀)**를 기준으로 하는 **절대 단위**이다. 디바이스 해상도나 브라우저 설정에 관계없이 항상 고정된 크기를 유지하므로, 정밀한 픽셀 단위 제어가 필요할 때 사용한다.

### **em (Equal to Element)**
**부모 요소 또는 현재 요소**의 `font-size`를 기준으로 하는 **상대 단위**이다. 폰트 크기뿐만 아니라 여백(padding, margin) 등에도 적용할 수 있으며, 주변 컨텍스트와의 비례 관계를 설정할 때 유용하다.

### **rem (Root em)**
**최상위 요소(`<html>`)**의 `font-size`를 기준으로 하는 **상대 단위**이다. 계층 구조에 영향을 받지 않고 전역적인 일관성을 유지하기 위해 사용된다.

---

## 2. 비교 분석

| 구분 | px | em | rem |
| :--- | :--- | :--- | :--- |
| **기준** | 고정된 1픽셀 | **부모** 또는 현재 요소의 폰트 크기 | **루트(`<html>`)** 요소의 폰트 크기 |
| **성격** | 절대적 (Absolute) | 상대적 (Relative) | 상대적 (Relative) |
| **장점** | 직관적이고 정밀한 설계 가능 | 컴포넌트 내부 비례 조절에 용이 | 전역적 일관성 유지 및 접근성 우수 |
| **단점** | 사용자 설정에 따른 가변성 부족 | 계층이 깊어지면 크기 계산이 복잡함 | 특정 컴포넌트 내의 비례 조정은 어려움 |

## 3. 심화 개념 보강: 언제 무엇을 쓸까?

### **① 왜 rem이 권장되나요? (접근성)**
현대 웹에서는 `px`보다 `rem` 사용을 더 권장한다. 사용자가 브라우저 설정에서 "글자 크기 크게"를 선택하면, `rem`은 루트 폰트 크기를 따라가며 화면 전체를 적절히 키워주지만, `px`은 요지부동이기 때문이다.

### **② em이 빛을 발하는 순간: 모듈화**
질문하신 내용처럼 `em`은 특정 컴포넌트 내부의 **비례 관계**를 설계할 때 매우 강력하다.
* **예시**: 버튼 컴포넌트에서 `font-size`를 `1em`, `padding`을 `0.5em`으로 설정하면, 나중에 버튼의 글자 크기만 키워도 패딩이 그에 맞춰 자동으로 늘어난다.

### **③ px을 사용해도 괜찮은 경우**
* 테두리 두께 (`border-width`): 1px 선이 사용자 설정에 따라 2px, 3px로 굵어지는 것은 원치 않는 경우가 많다.
* 크기가 절대적으로 고정되어야 하는 아이콘이나 로고 이미지 등.
