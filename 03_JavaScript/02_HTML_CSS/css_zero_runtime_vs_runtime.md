## 1. 개념 정의

### **제로 런타임 CSS란?**
CSS-in-JS의 편리한 개발 경험(DX)을 유지하면서도, 스타일 계산을 브라우저(Runtime)가 아닌 **빌드 시점(Build time)**에 완료하여 정적 CSS 파일을 생성하는 방식이다.

* **Runtime CSS-in-JS (예: Styled-components)**: 앱 실행 중 자바스크립트가 스타일을 계산하고 `<style>` 태그를 DOM에 삽입한다.
* **Zero-runtime CSS (예: Vanilla Extract, Panda CSS)**: 빌드 도구가 코드를 분석하여 미리 CSS 파일을 추출하므로, 브라우저에서는 추가적인 자바스크립트 연산이 필요 없다.

---

## 2. 비교 분석: 일반 CSS-in-JS vs 제로 런타임 CSS

| 구분 | Runtime CSS-in-JS | Zero-runtime CSS |
| :--- | :--- | :--- |
| **스타일 생성 시점** | 브라우저 실행 시 (동적) | 프로젝트 빌드 시 (정적) |
| **런타임 오버헤드** | 존재 (JS가 스타일 계산 및 주입) | **없음** (순수 CSS 파일 로드) |
| **캐싱 효율** | 낮음 (동적 생성으로 인한 제약) | **높음** (브라우저가 CSS 파일 캐싱 가능) |
| **파일 크기** | 런타임 라이브러리 포함 필요 | 라이브러리 미포함 (CSS 파일만 남음) |
| **동적 스타일링** | 매우 쉬움 (Props 기반 제어) | 상대적으로 까다로움 (CSS 변수 활용 필요) |

---

## 3. 심화 개념 보강

### **① 왜 '제로(Zero)'인가?**
"제로 런타임"이라는 이름은 브라우저가 자바스크립트를 실행하여 스타일을 만드는 과정이 **0(Zero)**이라는 뜻이다. 이는 메인 스레드의 점유 시간을 줄여 **LCP(Largest Contentful Paint)**나 **INP(Interaction to Next Paint)** 같은 성능 지표를 개선하는 데 직접적인 도움을 준다.

### **② 동적 스타일링의 한계 극복 (CSS Variables)**
빌드 시점에 스타일이 확정되어야 하므로, 사용자 입력에 따라 실시간으로 변하는 스타일(예: 마우스 위치에 따른 배경색)을 구현하기 어렵다는 단점이 있다. 
* **해결책**: 이를 보완하기 위해 최신 도구들은 **CSS 변수(Custom Properties)**를 활용한다. 정적 CSS 틀은 유지하되, 변수 값만 자바스크립트로 업데이트하여 런타임 성능 저하를 최소화하면서 동적 기능을 구현한다.

### **③ 대표적인 라이브러리**
* **Vanilla Extract**: TypeScript를 사용하여 타입 안정성이 뛰어난 CSS를 빌드 시점에 추출한다.
* **Panda CSS**: 최신 기술인 타입 안정성과 고성능을 결합한 스타일 엔진이다.
* **Linaria**: 익숙한 Styled-components 문법을 사용하면서도 결과물은 정적 CSS로 만들어준다.
