## 1. 개념 정의

### **시맨틱 마크업 (Semantic Markup)**

HTML 태그가 단순히 화면에 어떻게 보일지(Presentation)가 아니라, 해당 콘텐츠가 **어떤 의미(Meaning)**를 가지고 있는지에 따라 적절한 태그를 선택하여 작성하는 기법이다.

* **비시맨틱 태그**: `<div>`, `<span>` (의미 없음, 단순히 구역을 나눔)
* **시맨틱 태그**: `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>` 등 (태그만 봐도 구조를 짐작 가능)

---

## 2. 비교 분석: 비시맨틱 vs 시맨틱

| 구분 | 비시맨틱 (`<div>` 위주) | 시맨틱 (적재적소 태그) |
| --- | --- | --- |
| **가독성** | 코드 지옥(Div Soup) 발생, 구조 파악 어려움 | 코드만으로 페이지 레이아웃 직관적 파악 가능 |
| **SEO** | 검색 엔진이 콘텐츠의 중요도를 판단하기 힘듦 | 핵심 콘텐츠와 부가 정보를 명확히 구분 |
| **웹 접근성** | 스크린 리더가 각 구역의 역할을 인지 못 함 | 시각 장애인이 문서 구조를 논리적으로 탐색 가능 |
| **유지보수** | 클래스명(`class="header"`)에 의존해야 함 | 표준화된 태그로 협업 및 수정이 용이함 |

---

## 3. 심화 개념 보강

### **① CSR 환경에서의 SEO와 시맨틱 마크업**

CSR 환경(React, Vue 등)에서는 초기 HTML이 비어있어 SEO에 불리하다는 인식이 있지만, 시맨틱 마크업은 여전히 필수적이다.

* **Googlebot의 진화**: 구글 크롤러는 자바스크립트를 실행하여 렌더링된 결과를 수집한다. 이때 시맨틱 태그가 잘 갖춰져 있다면, 크롤러는 "이 부분이 실제 본문(article)이구나"를 더 정확히 판단한다.
* **정보의 위계**: `<h1>`부터 `<h6>`까지의 순서, `<nav>`의 유무 등은 페이지의 주제를 파악하는 지표가 된다.

### **② 웹 접근성(A11y)과 랜드마크**

시맨틱 태그는 브라우저에게 '랜드마크' 역할을 한다.

* 사용자가 키보드 단축키를 통해 "메뉴로 건너뛰기"나 "본문으로 바로 가기"를 할 때, 브라우저는 `<nav>`나 `<main>` 태그를 기준으로 작동한다. `<div>`만으로는 이러한 사용자 편의를 제공할 수 없다.
