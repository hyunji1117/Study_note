## 🏗️ 자바스크립트 호이스팅 (Hoisting)

호이스팅은 자바스크립트 엔진이 코드를 실행하기 전, **변수와 함수의 선언을 코드의 최상단으로 끌어올리는 것처럼** 동작하는 특징을 말한다.

### 1. 선언 방식에 따른 차이

| 구분 | `var` | `let` / `const` | 함수 선언식 |
| --- | --- | --- | --- |
| **초기화 시점** | 선언 시 즉시 `undefined`로 초기화 | 실제 코드 라인에 도달했을 때 초기화 | 함수 전체가 즉시 메모리에 할당 |
| **선언 전 접근** | 가능 (`undefined` 반환) | **불가능 (`ReferenceError`)** | 가능 (정상 실행) |
| **할당 호이스팅** | 할당은 끌어올려지지 않음 | 할당은 끌어올려지지 않음 | 해당 없음 (통째로 호이스팅) |

---

### 2. TDZ (Temporal Dead Zone, 일시적 사각지대)

`let`과 `const`로 선언된 변수가 호이스팅은 되지만 사용할 수 없는 이유는 바로 **TDZ** 때문이다.

* **정의:** 스코프의 시작 지점부터 변수의 선언(및 초기화) 지점까지의 구간을 의미한다.
* **동작:** `var`와 달리 `let`은 선언 단계와 초기화 단계가 분리되어 실행된다. 선언은 호이스팅되어 인지되지만, 실제 코드에서 초기화 단계에 도달하기 전까지 변수는 **TDZ**에 갇혀 접근 시 에러가 발생한다.
* **장점:** 변수가 선언되기 전에 사용되는 것을 방지하여 코드의 예측 가능성을 높이고 잠재적인 버그를 줄여준다.
