## 1. 개념 정의

### **접근 제어자란?**
클래스 내부의 변수(필드)나 함수(메서드)에 대해 **접근할 수 있는 범위**를 제한하는 문법이다. 이를 통해 내부 로직을 숨기고 외부에서의 잘못된 조작을 방지하는 **캡슐화(Encapsulation)**를 실현한다. 

* **Public**: 어디서든 접근 가능 (기본값).
* **Private**: 해당 클래스 내부에서만 접근 가능.
* **Protected**: 해당 클래스와 이를 상속받은 자식 클래스에서만 접근 가능.

---

## 2. 비교 분석: JavaScript vs TypeScript

| 구분 | JavaScript (ES2022+) | TypeScript |
| :--- | :--- | :--- |
| **Public** | 기본 동작 (별도 키워드 없음) | `public` 키워드 지원 (명시적 선언) |
| **Private** | `#` 접두사 사용 (언어 차원 강제) | `private` 키워드 및 `#` 모두 지원 |
| **Protected** | **지원 안 함** (`_` 컨벤션 사용) | `protected` 키워드로 **언어 차원 지원** |
| **컴파일 시점** | 런타임에서 실제로 접근 차단 | 컴파일 단계에서 체크 (런타임에선 제거됨) |
| **편의 기능** | 없음 | **파라미터 프로퍼티**로 자동 초기화 지원 |

---

## 3. 심화 개념 보강

### **① JavaScript의 # (Private Identifier)**
자바스크립트의 `#`은 단순히 문법적 설탕이 아니라 **런타임에서 실제로 데이터가 보호**된다. 클래스 외부에서 `#` 멤버에 접근하려고 하면 즉시 구문 오류(Syntax Error)가 발생하며, 이는 클로저와 유사한 수준의 강력한 은닉화를 제공한다.

### **② TypeScript의 protected**
자바스크립트에서는 관습적인 `_`에 의존해야 했지만, 타입스크립트의 `protected`는 **상속 관계에서의 캡슐화**를 완벽히 구현한다.
* 외부 객체를 통한 접근(`instance.c`)은 막고,
* 자식 클래스 내부의 메서드(`this.c`)를 통한 접근은 허용하여 확장성 있는 설계를 돕는다.

### **③ 파라미터 프로퍼티 (Parameter Properties)**
타입스크립트의 백미로, 생성자 매개변수에 접근 제어자를 붙여 **선언과 초기화를 한 번에** 처리하는 문이다.

```typescript
// 일반적인 방식
class User {
  private name: string;
  constructor(name: string) { this.name = name; }
}

// 파라미터 프로퍼티 (간결함!)
class User {
  constructor(private name: string) {}
}
```
