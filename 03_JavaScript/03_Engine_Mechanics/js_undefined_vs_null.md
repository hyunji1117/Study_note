# undefined vs null
## 자바스크립트에서 두 값은 모두 "값이 없음"을 나타내지만, **발생 원인**과 **의도**에서 큰 차이가 있다.

### 1. 개념적 차이

| 구분 | undefined | null |
| --- | --- | --- |
| **의미** | 값이 할당되지 않은 상태 (고유의 기본값) | 의도적으로 값을 비워둔 상태 (명시적 부재) |
| **할당 주체** | 자바스크립트 엔진 (자동) | 개발자 (수동) |
| **타입 (typeof)** | `undefined` | `object` (자바스크립트 설계상 오류) |

### 2. 비교 연산

* **느슨한 비교 (`==`):** `null == undefined` 👉 **true** (둘 다 빈 값으로 취급)
* **엄격한 비교 (`===`):** `null === undefined` 👉 **false** (타입이 다르기 때문)

### 3. 메모리 관리와 가비지 컬렉션 (GC)

* **null:** 특정 객체에 대한 참조를 명시적으로 끊을 때 사용한다. 변수에 `null`을 할당하면 이전에 참조하던 객체는 참조 카운트가 0이 되어 **가비지 컬렉션의 대상**이 된다. 즉, 메모리 해제를 유도하는 도구로 쓰인다.
* **undefined:** 단지 선언만 되고 초기화되지 않았음을 나타낼 뿐이며, 그 자체로 메모리 해제를 유도하는 메커니즘을 가지고 있지 않는다.
