# 웹 접근성 정리 (Frontend 관점)

## 1. 웹 접근성의 개념

웹 접근성이란 장애인, 고령자 등 **신체적 제약이 있는 사용자**를 포함하여 **모든 사용자**가 웹 콘텐츠를 동등하게 이용할 수 있도록 보장하는 것을 의미한다.

* **디지털 경사로**: 건물 입구의 휠체어 경사로처럼, 디지털 세상에서도 정보 취득의 장벽을 제거하는 것이다.
* **보편적 설계**: 특정 대상뿐만 아니라 저사양 기기, 느린 네트워크, 눈부신 야외 환경 등 다양한 상황의 모든 사용자를 고려한다.

---

## 2. 주요 개선 방법 (Technical Implementation)

### 시맨틱 마크업 (Semantic HTML)

* `<div>`나 `<span>` 대신 `<header>`, `<nav>`, `<main>`, `<footer>` 등 **의미가 담긴 태그**를 사용한다.
* **효과**: 스크린 리더가 페이지의 구조를 명확히 해석하여 시각 장애 사용자에게 정확한 정보를 전달한다.

### WAI-ARIA 활용

* HTML 태그만으로 의미 전달이 부족한 동적 UI(탭 메뉴, 모달 등)에 `aria-label`, `role` 등 **ARIA 속성**을 추가한다.
* **효과**: 복잡한 상태 변화나 컴포넌트의 역할을 보조공학 기기에 알릴 수 있다.

### 키보드 접근성 (Keyboard Navigation)

* 모든 인터랙티브 요소(버튼, 링크 등)에 `focus`가 잡히도록 구현하고, 시각적으로 포커스 위치를 표시한다.
* **효과**: 마우스 사용이 어려운 사용자가 `Tab` 키만으로 서비스를 이용할 수 있게 한다.

---

## 3. 현실적인 접근 방식과 한계

### 완벽함보다는 사용자 경험 중심

* 모든 장애 유형을 100% 만족시키는 것은 현실적으로 매우 어렵다.
* **사용자 여정 파악**: 개발자가 직접 스크린 리더를 켜고 서비스를 탐색하며 실제 겪는 불편함을 파악하는 것이 우선이다.

### 단계적 개선 (Iterative Improvement)

* 단순히 체크리스트 항목을 채우는 것에 매몰되지 않아야 한다.
* 기업의 리소스와 서비스 특성에 맞춰 **우선순위**를 정하고 점진적으로 개선해 나가는 것이 지속 가능한 방법이다.
