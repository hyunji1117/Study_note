## 🖥️ 멀티 프로세스 vs 멀티 스레드

컴퓨터 자원을 효율적으로 활용하여 작업의 안정성을 높이거나 처리 속도를 개선하기 위한 두 가지 핵심 기법이다.

### 1. 멀티 프로세스 (Multi-process)

하나의 프로그램이 여러 개의 독립적인 프로세스를 생성하여 실행되는 구조다.

* **독립적 메모리 공간**: 각 프로세스는 자신만의 메모리 영역을 할당받아 실행된다.
* **높은 안정성**: 프로세스 간 격리가 확실하여, 하나의 프로세스에 오류가 발생해도 전체 시스템이나 다른 프로세스에 영향을 주지 않는다.
* **브라우저 예시**: 크롬 브라우저의 각 **탭**은 별도의 프로세스로 동작합니다. 특정 탭이 멈추더라도 브라우저 전체가 종료되지 않는 이유이다.
* **단점**: 프로세스 생성 및 전환 시 자원 소모가 크며, 프로세스 간 통신(IPC)이 복잡하고 비용이 높다.

---

### 2. 멀티 스레드 (Multi-thread)

하나의 프로세스 내부에서 여러 개의 스레드가 동시에 작업을 수행하는 구조이다.

* **자원 공유**: 스레드들은 프로세스의 메모리 공간과 시스템 자원을 공유하며 실행된다.
* **높은 효율성**: 스레드 간 통신이 빠르고, 컨텍스트 스위칭 비용이 프로세스보다 저렴하여 가볍고 빠른 작업 분할이 가능하다.
* **주의사항**: 여러 스레드가 동시에 동일한 공유 자원에 접근할 때 데이터 충돌이 발생할 수 있습니다. 이를 관리하기 위해 **동기화(Synchronization)** 처리가 필요하며, 이 과정에서 코드 복잡도가 높아질 수 있다.

---

### 3. 한눈에 비교하기

| 구분 | 멀티 프로세스 | 멀티 스레드 |
| --- | --- | --- |
| **자원 공유** | 공유하지 않음 (독립적) | 프로세스 내 자원을 공유함 |
| **안정성** | 높음 (개별 격리) | 낮음 (하나의 스레드 장애가 프로세스 전체 영향) |
| **통신 비용** | 높음 (IPC 필요) | 낮음 (공유 메모리 활용) |
| **주요 사례** | 크롬 탭, 멀티 태스킹 앱 | 게임 엔진, 서버 요청 처리 |
