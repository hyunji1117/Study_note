## 1. 개념 정의

### **Web Storage API**
브라우저에서 키-값(`key-value`) 쌍으로 데이터를 저장할 수 있게 해주는 API이다. 쿠키(Cookie)보다 저장 용량이 크고(약 5MB), 서버로 매번 전송되지 않아 성능상 이점이 있다.

* **localStorage**: 데이터를 브라우저에 **반영구적**으로 저장한다. 브라우저를 닫거나 컴퓨터를 재부팅해도 데이터가 유지된다.
* **sessionStorage**: 데이터의 수명이 **페이지 세션**이 유지되는 동안으로 제한된다. 탭이나 창을 닫으면 데이터가 즉시 삭제된다.

---

## 2. 비교 분석: 저장소별 특징

| 구분 | localStorage | sessionStorage |
| :--- | :--- | :--- |
| **데이터 유지** | 브라우저 종료 후에도 유지 (영구적) | 탭/창 종료 시 삭제 (일시적) |
| **공유 범위** | 동일 도메인 내 모든 탭/창에서 공유 | 같은 도메인이라도 **탭 단위**로 독립적 |
| **저장 용량** | 약 5MB ~ 10MB (브라우저별 상이) | 약 5MB |
| **주요 용도** | 자동 로그인, 테마 설정(다크모드), 장바구니 | 입력 폼 임시 저장, 일회성 인증 정보 |

## 3. 심화 개념 보강: 보안 이슈와 해결책

Web Storage는 편리하지만, 자바스크립트로 쉽게 접근할 수 있어 **XSS(Cross-Site Scripting)** 공격에 취약하다는 치명적인 단점이 있다.

### **① 보안 취약점**
* **공격 경로**: 악성 스크립트가 실행되면 `window.localStorage`를 통해 저장된 모든 데이터를 탈취할 수 있다.
* **민감 데이터 노출**: 사용자 비밀번호나 개인정보를 평문으로 저장할 경우, 물리적 접근이나 악성 스크립트에 의해 정보가 유출될 위험이 크다.

### **② 권장 해결책: HttpOnly Cookie**
민감한 인증 정보(Access Token 등)를 다룰 때는 저장소 대신 **쿠키**를 활용하는 것이 정석이다.

* **HttpOnly 플래그**: 자바스크립트(`document.cookie`)에서 쿠키에 접근하는 것을 차단하여 XSS를 통한 토큰 탈취를 방지한다.
* **Secure 플래그**: HTTPS 프로토콜을 통해서만 쿠키가 전송되도록 제한한다.

> **💡 요약 가이드**
> 비민감하고 설정 위주의 데이터는 **localStorage**, 보안이 중요하거나 세션 내에서만 유효한 데이터는 **HttpOnly 쿠키** 혹은 **sessionStorage**를 상황에 맞춰 선택이 필요다.
