## 🚀 HTTP/2: 성능과 효율의 혁신

HTTP/2는 전송 효율을 극대화하여 웹 페이지 로드 속도를 높이고 네트워크 자원 낭비를 줄이는 데 집중한 프로토콜이다.

### 1. 핵심 특징

* **멀티플렉싱 (Multiplexing):** 하나의 TCP 연결 내에서 여러 개의 요청과 응답을 독립적인 **스트림(Stream)** 단위로 쪼개어 동시에 병렬 처리한다. HTTP/1.1의 순차적 처리 한계를 극복하여 리소스 로딩 속도를 비약적으로 향상시켰다.
* **헤더 압축 (Header Compression):** **HPACK** 압축 방식을 도입하여 매 요청마다 중복되는 헤더 정보를 압축하고, 허프만 코딩(Huffman Coding)을 통해 전송 데이터량을 최소화한다.
* **서버 푸시 (Server Push):** 클라이언트가 요청하지 않아도 서버가 HTML 분석 후 필요한 리소스(CSS, JS 등)를 미리 보내주는 기능이다. 이를 통해 클라이언트의 추가 요청 대기 시간을 줄여 초기 렌더링 속도를 개선한다.

---

### 2. HTTP/1.1 Pipelining과의 차이점

| 구분 | HTTP/1.1 Pipelining | HTTP/2 Multiplexing |
| --- | --- | --- |
| **처리 방식** | 응답을 기다리지 않고 요청을 연달아 보냄 | 요청과 응답을 잘게 쪼개어 완전히 병렬 처리 |
| **응답 순서** | **반드시 요청 순서대로** 응답해야 함 | 순서와 상관없이 먼저 처리된 것부터 응답 가능 |
| **고질적 문제** | **HOL Blocking** (앞선 응답이 지연되면 뒤도 지연됨) | HOL Blocking 문제 해결 (병렬 전송) |
