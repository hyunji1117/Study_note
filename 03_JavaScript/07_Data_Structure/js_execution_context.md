## 🏗️ 실행 컨텍스트 (Execution Context)

실행 컨텍스트는 자바스크립트 엔진이 코드를 실행하기 위해 필요한 환경(변수, 함수 선언, 스코프 등)을 정의하고 관리하는 추상적인 개념이다. 

### 1. 실행 컨텍스트의 종류

* **전역 실행 컨텍스트 (Global Execution Context):** 자바스크립트 파일이 처음 실행될 때 생성된다. 프로그램 종료 시까지 유지되며, 싱글 스레드 특성상 단 하나만 존재한다.
* **함수 실행 컨텍스트 (Function Execution Context):** 함수가 호출될 때마다 독립적으로 생성된다. 함수 실행이 완료되면 해당 컨텍스트는 파기된다.

---

### 2. 실행 컨텍스트의 3대 구성 요소

자바스크립트 엔진은 코드를 실행하기 위해 다음 세 가지 정보를 컨텍스트 내에 담아둔다.

1. **변수 객체 (Variable Object):** * 컨텍스트 내의 변수, 매개변수(arguments), 함수 선언을 저장하는 공간이다.
* 전역 컨텍스트에서는 **전역 객체**가, 함수 컨텍스트에서는 **활성 객체(Activation Object)**가 이 역할을 수행한다.


2. **스코프 체인 (Scope Chain):** * 현재 컨텍스트와 상위(외부) 렉시컬 환경의 연결 고리이다.
* 특정 변수를 찾을 때 현재 스코프에 없으면 이 체인을 따라 상위 환경으로 올라가며 탐색한다.


3. **this 바인딩:** * 컨텍스트에 따라 `this`가 가리키는 객체가 결정된다.
* 전역에서는 전역 객체를, 함수 내에서는 호출 방식에 따라 참조 대상이 달라진다.
