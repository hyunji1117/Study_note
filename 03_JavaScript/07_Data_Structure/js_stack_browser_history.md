브라우저의 **뒤로가기/앞으로가기** 로직을 스택(Stack) 자료구조 활용

---

## 🌐 스택을 활용한 브라우저 히스토리 구현

프론트엔드에서 브라우저의 탐색 기록을 관리할 때는 **LIFO(Last In, First Out)** 구조인 스택을 활용하는 것이 가장 효율적입니다. 이를 위해 두 개의 독립적인 스택이 필요합니다.

### 1. 필요한 자료구조

* **`backStack` (뒤로가기 스택):** 사용자가 현재까지 방문한 페이지들을 순서대로 저장합니다. 이 스택의 가장 위에 있는(`top`) 데이터가 **현재 보고 있는 페이지**가 됩니다.
* **`forwardStack` (앞으로가기 스택):** '뒤로가기'를 실행했을 때, 다시 앞으로 돌아갈 수 있도록 이전에 방문했던 페이지들을 임시 보관합니다.

---

### 2. 동작 흐름 (Algorithm)

#### ① 새로운 페이지 방문 시 (`New Page Visit`)

사용자가 링크를 클릭하거나 주소를 입력해 새로운 페이지로 이동하는 경우입니다.

* **Action:** 현재 페이지를 `backStack`에 **push**합니다.
* **State:** 이전의 앞으로 가기 경로는 더 이상 유효하지 않으므로 `forwardStack`을 **비웁니다(Clear)**.

#### ② 뒤로가기 버튼 클릭 시 (`Go Back`)

사용자가 이전 페이지로 돌아가고 싶을 때의 로직입니다.

* **Action:** `backStack`에서 현재 페이지를 **pop**하여 `forwardStack`에 **push**합니다.
* **Result:** `backStack`의 새로운 **top**에 위치한 페이지가 화면에 출력되는 현재 페이지가 됩니다.

#### ③ 앞으로가기 버튼 클릭 시 (`Go Forward`)

뒤로가기를 한 후, 다시 원래의 페이지로 이동하고 싶을 때입니다.

* **Action:** `forwardStack`에서 페이지를 **pop**하여 `backStack`에 다시 **push**합니다.
* **Result:** `backStack`의 가장 위에 쌓인(`top`) 페이지가 현재 페이지가 됩니다.

---

### 3. 요약 테이블

| 동작 | `backStack` 변화 | `forwardStack` 변화 |
| --- | --- | --- |
| **새 페이지 방문** | 현재 페이지 **Push** | **초기화 (Empty)** |
| **뒤로가기** | Pop (기존 현재 페이지 제거) | **Push** (제거된 페이지 추가) |
| **앞으로가기** | **Push** (복구된 페이지 추가) | Pop (기존 데이터 제거) |

---

> **💡 추가 팁:**
> 실제 JavaScript의 `History API`(`window.history.back()`, `window.history.forward()`)도 내부적으로 이와 유사한 스택 구조로 관리된다.
