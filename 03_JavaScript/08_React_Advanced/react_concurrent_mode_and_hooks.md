## 1. 개념 정의

### **동시성 모드 (Concurrent Mode)란?**
리액트가 여러 상태 업데이트를 **동시에 관리**하며, 작업의 **우선순위**에 따라 렌더링 순서를 유연하게 조정하는 기능이다.

* **과거 (Stack Rendering)**: 한 번 시작한 렌더링은 끝날 때까지 브라우저의 메인 스레드를 점유하여 사용자 입력을 차단함 (Blocking).
* **현재 (Concurrent Rendering)**: 렌더링 중간에 더 중요한 작업(타이핑, 클릭 등)이 들어오면 기존 작업을 일시 중단하고 중요한 작업을 먼저 처리한 뒤, 나중에 다시 돌아와 남은 작업을 마무리함 (Interruptible).

## 2. 주요 기능 및 비교

| 구분 | startTransition | useDeferredValue |
| :--- | :--- | :--- |
| **대상** | 상태 업데이트 로직 (함수) | 상태 값 (데이터) |
| **역할** | UI 업데이트를 '전환(Transition)'으로 분류하여 우선순위를 낮춤 | 빈번하게 발생하는 값의 업데이트를 뒤로 미룸 |
| **사용 예시** | 탭 전환, 검색 결과 필터링 실행 | 검색어 입력값에 따른 리스트 렌더링 지연 |
| **특징** | `isPending` 상태를 통해 로딩 표시 가능 | 이전 값을 유지하다가 여유가 생기면 새 값으로 업데이트 |

## 3. 심화 개념 보강

### **① 우선순위 기반 스케줄링**
동시성 모드의 핵심은 모든 업데이트가 평등하지 않다는 점이다.
* **Urgent Updates**: 타이핑, 클릭, 누르기 등 즉각적인 피드백이 필요한 직관적인 상호작용.
* **Transition Updates**: 하나의 뷰에서 다른 뷰로의 전환 등 사용자가 약간의 지연을 예상할 수 있는 작업.

### **② 렌더링 중단과 재개 (Interruptible Rendering)**
리액트는 렌더링을 시작하기 전에 "이 작업이 브라우저를 얼마나 오래 붙잡고 있을까?"를 계산하지 않는다. 대신 작업을 아주 작은 단위로 나누어 수행하며, 매 단위마다 메인 스레드에 더 급한 일이 있는지 확인한다. 만약 있다면 현재 작업을 메모리에 **'스냅샷'**으로 남겨두고 긴급한 일을 먼저 처리한다.

### **③ 사용자 경험(UX) 최적화: 응답성 유지**
무거운 차트를 그리거나 수만 개의 리스트를 필터링할 때 화면이 멈추는 현상을 방지한다. 사용자는 키보드 입력을 계속할 수 있고, 리액트는 백그라운드에서 조용히 결과물을 준비하다가 계산이 끝나면 화면에 '짠'하고 나타낸다.
