## 폼 데이터 관리: 제어 vs 비제어 컴포넌트

리액트는 폼의 입력값을 리액트 시스템(State) 안에서 관리할 것인지, 아니면 브라우저의 기본 동작(DOM)에 맡길 것인지에 따라 두 가지 방식으로 나뉜다.

### 1. 제어 컴포넌트 (Controlled Component)

* **정의**: 리액트의 **상태(State)**를 통해 입력값을 제어하는 방식이다.
* **동작**: 입력 요소의 `value`를 리액트 상태와 동기화하며, 사용자가 값을 입력할 때마다 `onChange` 이벤트 핸들러를 통해 상태를 즉시 업데이트한다.
* **장점**: 리액트 상태로 값이 관리되므로 입력값의 실시간 검증(Validation), 조건에 따른 입력 제한, 복잡한 폼 로직 처리에 매우 유용하다.

### 2. 비제어 컴포넌트 (Uncontrolled Component)

* **정의**: 입력값을 리액트 상태로 관리하지 않고, **DOM**에서 직접 관리하는 방식이다.
* **동작**: 리액트가 값을 제어하지 않으며, 필요할 때 **`useRef`**를 통해 생성된 `ref` 객체로 DOM 요소에 직접 접근하여 값을 읽어온다.
* **장점**: 상태 업데이트에 따른 리렌더링 비용이 발생하지 않아 상대적으로 성능 면에서 유리하며, 간단한 구조의 폼을 구현할 때 코드가 간결해진다.

---

### 3. 상황별 선택 가이드

| 상황 | 추천 방식 | 이유 |
| --- | --- | --- |
| **실시간 유효성 검사** | **제어 (Controlled)** | 입력 시마다 상태를 확인하여 즉각적인 피드백 가능 |
| **단순 제출용 폼** | **비제어 (Uncontrolled)** | 제출 시점에만 `ref`로 값을 가져오면 되므로 효율적 |
| **조건부 입력 활성화** | **제어 (Controlled)** | 특정 입력값에 따라 다른 필드를 노출/차단하기 용이함 |
| **성능 최적화가 중요한 대형 폼** | **비제어 (Uncontrolled)** | 잦은 타이핑으로 인한 전체 리렌더링 방지 |
