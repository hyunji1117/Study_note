## ⚡ 리액트 성능 최적화 전략

성능 최적화의 핵심은 불필요한 리렌더링을 방지하고, 리소스 로딩 효율을 극대화하는 것이다.

### 1. 메모이제이션 (Memoization)

컴포넌트나 계산된 값, 함수를 메모리에 저장해 두었다가 재사용하는 기법이다.

`React.memo`: 컴포넌트 단위의 메모이제이션이다. 부모 컴포넌트가 리렌더링되더라도 `props`가 변경되지 않았다면 자식 컴포넌트의 리렌더링을 방지한다. 렌더링 비용이 큰 컴포넌트에서 효과적이다.
`useMemo`: 복잡한 계산 결과값을 메모이제이션한다. 의존성 배열의 값이 변경되지 않으면 이전 계산값을 재사용하여 불필요한 재계산을 막는다.
`useCallback`: 함수 자체를 메모이제이션한다. 함수가 재생성되는 것을 방지하여, 해당 함수를 `props`로 받는 자식 컴포넌트의 불필요한 리렌더링을 차단한다.

---

### 2. 코드 스플리팅 (Code Splitting)

하나의 큰 번들 파일을 여러 개의 작은 청크(Chunk)로 나누어 필요한 시점에 로드하는 방식이다.

* **사용 목적**: 애플리케이션의 규모가 커짐에 따라 비대해진 초기 로드 파일을 분산시켜 초기 로딩 속도를 개선한다.
* **구현 방법**: `React.lazy`와 `Suspense`를 사용하여 컴포넌트를 동적으로 로드한다.
* **적용 사례**:
* **초기 로딩 개선**: 핵심 코드만 먼저 로드하고 나머지는 필요할 때 불러온다.
* **라우트별 분할**: 각 페이지(Route)별로 필요한 코드만 분리하여, 사용자가 해당 주소에 접근할 때만 로드하도록 설정한다.
