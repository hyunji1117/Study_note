## ⚛️ React 데이터 관리: Props와 State

리액트 컴포넌트는 데이터를 다루기 위해 두 가지 핵심 객체를 사용한다. 이들의 차이점을 이해하는 것이 효율적인 컴포넌트 설계의 시작이다.

### 1. Props (Properties)

* **정의**: 부모 컴포넌트가 자식 컴포넌트에 전달하는 데이터이다.
* **불변성 (Read-only)**: `props`는 읽기 전용이며, 자식 컴포넌트에서 직접 수정할 수 없다.
* **목적**: 컴포넌트 간 데이터 흐름을 예측 가능하게 만들고, 재사용성을 높이는 데 기여한다.
* **특징**: 단방향 데이터 흐름 원칙에 따라 부모에서 자식으로만 전달된다. 이로 인해 상태 변화의 근원을 파악하기 쉬워 디버깅이 용이하고 코드의 캡슐화가 강화된다.

### 2. State

* **정의**: 컴포넌트 내부에서 스스로 관리하는 동적인 데이터이다.
* **가변성**: 사용자 입력이나 네트워크 응답 등에 따라 동적으로 변경될 수 있다.
* **영향**: `state`가 변경되면 해당 컴포넌트는 **리렌더링**되며 UI가 최신 상태로 업데이트된다.
* **활용**: 주로 컴포넌트 내에서 유지하고 변화시켜야 하는 로컬 데이터를 관리할 때 사용된다.

---

### 3. 데이터 흐름 제어: 상태 끌어올리기 (Lifting State Up)

자식 컴포넌트가 부모로부터 받은 `props`를 변경해야 할 상황이 생기면 직접 수정하는 대신 **상태 끌어올리기** 기법을 사용한다.

* **방법**: 부모 컴포넌트에서 상태(`state`)를 관리하고, 이 상태를 변경할 수 있는 **함수**를 자식에게 `props`로 전달한다.
* **결과**: 데이터는 여전히 부모에서 자식으로 흐르는 단방향성을 유지하면서, 자식은 부모의 상태를 간접적으로 변경하여 일관성을 유지할 수 있다.
