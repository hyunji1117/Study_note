## 1. 개념 정의

### **useEffect** (Component Did Mount/Update/Will Unmount)

컴포넌트의 렌더링 결과가 **브라우저에 실제로 반영된 후(Paint)** 실행되는 Hook이다. 비동기적으로 동작하며, 사용자 경험을 방해하지 않아야 하는 작업에 적합하다.

### **useLayoutEffect** (DOM Mutation 후, Paint 전)

컴포넌트가 렌더링되고 **DOM은 변경되었지만, 브라우저가 화면을 그리기 전(Before Paint)**에 실행되는 Hook이다. 동기적으로 동작하며, 레이아웃 계산이 필요한 작업에 필수적이다.

---

## 2. 비교 분석

| 구분 | useEffect | useLayoutEffect |
| --- | --- | --- |
| **실행 타이밍** | 브라우저가 화면을 그린(Paint) **후** | 브라우저가 화면을 그리기 **전** |
| **실행 방식** | **비동기적 (Asynchronous)** | **동기적 (Synchronous)** |
| **주요 용도** | 데이터 페칭, 이벤트 리스너, 로깅 등 | DOM 요소 크기/위치 측정, 스크롤 위치 조작 |
| **사용자 경험** | 화면 깜빡임이 발생할 수 있음 | 화면 깜빡임 없이 즉각 반영됨 |
| **성능 영향** | 렌더링 차단 없음 (성능 유리) | 내부 작업 완료 시까지 렌더링 차단 (주의 필요) |

---

## 3. 심화 개념 보강: 왜 '동기적' 실행이 중요한가?

React의 렌더링 라이프사이클을 이해하면 두 Hook의 차이가 더 명확해진다.

1. **React가 DOM 업데이트**: 내부적으로 가상 DOM을 실제 DOM에 반영한다.
2. **useLayoutEffect 실행**: 아직 화면은 이전 상태이다. 여기서 DOM의 `offsetHeight` 등을 측정하거나 상태를 변경하면, React는 **화면을 그리기 전에 다시 계산**을 수행한다.
3. **Browser Paint**: 최종 결과물이 화면에 그려진다.
4. **useEffect 실행**: 화면이 다 그려진 후, 여유로운 상태에서 비동기 작업을 처리한다.
