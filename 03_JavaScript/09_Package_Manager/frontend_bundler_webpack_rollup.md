## 1. 개념 정의

### **번들러(Bundler)란?**
웹 애플리케이션을 구성하는 수많은 자원(JavaScript, CSS, 이미지 등)을 브라우저가 최적화된 상태로 로드할 수 있도록 **하나 또는 소수의 파일로 묶어주는 도구**이다.

---

## 2. 번들러가 필요한 이유

| 구분 | 주요 내용 | 기대 효과 |
| :--- | :--- | :--- |
| **네트워크 최적화** | 수백 개의 파일을 소수의 번들 파일로 통합 | HTTP 요청 횟수 감소 및 로딩 속도 개선 |
| **코드 최적화** | Tree Shaking을 통해 사용하지 않는 코드 제거 | 번들 크기 감소 및 런타임 성능 향상 |
| **브라우저 호환성** | 최신 문법(ES6+)을 구형 문법(ES5)으로 변환 | 다양한 사용자 환경에서 안정적 동작 보장 |
| **모듈 시스템 지원** | `import`/`export` 문법이 없던 환경에서도 모듈화 구현 | 코드의 재사용성 및 유지보수성 증대 |

---

## 3. 심화 개념 보강

### **① Tree Shaking (데드 코드 제거)**
Rollup이나 Webpack 같은 현대적 번들러의 핵심 기능이다. 나무를 흔들어 죽은 잎을 떨어뜨리듯, 실제로 **참조되지 않는 코드를 분석하여 최종 번들에서 제외**시키는 기술이다. 이는 특히 대규모 라이브러리를 사용할 때 용량을 획기적으로 줄여줍준다.

### **② 코드 분할 (Code Splitting)**
모든 코드를 하나의 커다란 `main.js`로 만드는 것이 항상 정답은 아니다. 번들러는 사용자가 현재 페이지에서 **당장 필요한 코드만 먼저 로드**하고, 나머지는 나중에 불러오는 전략을 가능하게 하여 초기 렌더링 속도(FCP)를 높힌다.

### **③ 에셋 관리 (Asset Management)**
JavaScript뿐만 아니라 CSS, Sass, 이미지, 폰트 등을 하나의 의존성 그래프 안에서 관리한다. 예를 들어, CSS 내부의 `url()` 경로를 번들링 시점에 자동으로 최적화된 경로로 치환해 주는 등의 작업을 수행한다.
